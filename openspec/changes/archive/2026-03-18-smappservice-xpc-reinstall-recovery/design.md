## 上下文

当前项目已经支持 SMAppService 与 SMJobBless 双安装路径，并在设置页提供安装/卸载入口。实际运行中，SMAppService 路径在 ad-hoc 更新后可能出现“helper 仍显示已安装但 XPC 不可达”的异常，用户会在路由开关或路由操作时直接收到通信错误，缺少就地恢复能力。

现有能力中，`PrivilegedHelperManager` 已提供可复用的卸载与安装流程，`RouterService` 负责路由请求与错误映射，`RouteListView` 与 `LegacyRouteListView` 负责向用户展示操作错误。本次变更应复用现有安装链路，在满足条件时引导用户执行“卸载后重装”恢复，而不是引入新的安装协议。

约束：
- 仅在 SMAppService 通道异常时触发，不影响 SMJobBless 既有行为。
- 不与“后台开关未授权”场景冲突；该场景继续沿用现有系统设置引导。
- 交互需兼容 macOS 12-14 代码结构（SwiftData 与 Core Data 双视图路径）。

## 目标 / 非目标

**目标：**
- 在 SMAppService 发生 XPC 通信失败时，提供清晰且可执行的恢复弹窗。
- 提供两个明确动作：`取消` 与 `自动重装`，其中 `自动重装` 为推荐操作。
- 自动重装复用现有 helper 卸载+安装能力，并在完成后刷新状态。
- 将触发条件限制为“SMAppService + XPC失败 + 非后台开关待授权”，避免误触发。

**非目标：**
- 不重构现有 helper 安装架构或替换 SMAppService/SMJobBless 策略。
- 不改变 Route 写入协议或 XPC 消息模型。
- 不在所有错误场景中统一弹恢复框（例如参数校验错误、路由写入业务失败）。

## 决策

### 决策 1：在 RouterService 暴露“可恢复错误信号”，由视图决定是否弹窗

在 `RouterService` 保留现有 `RouterError.xpcError` 的同时，新增可观察的恢复信号（例如 `@Published` 状态或专用上下文对象），包含是否满足“SMAppService 可重装恢复”条件。

理由：
- 错误识别应在服务层完成，避免 UI 重复判断安装方法和 pending 状态。
- 视图层仅处理展示与用户交互，职责清晰，便于 SwiftData/Core Data 两条路径复用。

备选方案：
- 在每个视图内解析 error.localizedDescription 决定是否弹窗：耦合高、脆弱且难以维护。

### 决策 2：自动重装动作复用 `uninstallHelper()` + `installHelper(method: .smAppService)`

恢复流程采用串行步骤：先卸载再安装；流程运行期间可临时禁用重复触发并在结束后刷新列表/状态。

理由：
- 复用已验证链路，减少新增系统权限风险。
- 与用户手动“卸载后安装”的现有心智一致，便于排障与日志追踪。

备选方案：
- 直接调用 `SMAppService.register()` 强制覆盖：无法覆盖旧注册状态/残留冲突，恢复确定性较差。

### 决策 3：弹窗动作语义采用“推荐主按钮 + 取消按钮”

在恢复弹窗中将 `自动重装` 放在主动作位置（默认确认），`取消` 作为 cancel 角色。

理由：
- 用户处于故障场景时通常希望尽快恢复，推荐动作可降低选择成本。
- 与需求“推荐方案2”保持一致。

备选方案：
- 两个同级按钮：表达不够明确，用户可能继续停留在错误状态。

### 决策 4：触发门槛包含“排除后台开关待授权场景”

只有当安装方法是 `smAppService`、XPC 请求失败且 `isPendingApproval == false` 时才触发自动重装弹窗。

理由：
- 待授权场景本质不是“需要重装”，应继续引导用户去系统设置开启后台权限。
- 防止错误恢复策略覆盖正确的权限引导路径。

## 风险 / 权衡

- [风险] 自动重装过程中再次失败（例如签名/权限问题） → [缓解] 保留原始错误提示并记录步骤级日志（卸载失败/安装失败分别提示）。
- [风险] 用户连续触发导致重复安装请求 → [缓解] 弹窗确认后进入一次性恢复流程，流程进行中禁用再次触发。
- [风险] 条件判断过宽导致误弹窗 → [缓解] 使用明确条件（activeMethod + pending 状态 + 错误类型）并补充回归测试场景。

## Migration Plan

1. 在 `RouterService` 增加 SMAppService XPC 失败的恢复信号与触发方法。
2. 在 `RouteListView` 与 `LegacyRouteListView` 接入恢复弹窗（取消/自动重装）。
3. 将自动重装动作编排为 `uninstallHelper()` 后 `installHelper(method: .smAppService)`。
4. 新增中英文本地化文案（标题、说明、取消、自动重装）。
5. 手动验证：
   - SMAppService + XPC 失败时弹窗出现；
   - 点击取消无副作用；
   - 点击自动重装后执行卸载再安装并刷新状态；
   - 后台开关关闭（pending）时不弹自动重装框。

回滚策略：若线上出现误触发或恢复失败率高，可先移除视图弹窗入口并保持原错误提示，不影响既有安装能力。
