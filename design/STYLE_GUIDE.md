# StaticRouteHelper UI Style Guide

> Last updated: 2026-03-24  
> Product: StaticRouteHelper (macOS / SwiftUI)

## 1. 设计方向

当前版本采用「专业工具型 + Swiss 风格克制表达」：

- 以信息可读性、操作效率为第一优先级。
- 视觉上保持轻量、干净、低装饰，不做高饱和大面积装饰。
- 使用单一主强调色 + 功能语义色（成功/警告/危险）表达状态。

## 2. 设计系统来源

- 设计检索技能：`ui-ux-pro-max`
- 栈约束：`swiftui`
- 当前实现以代码为最终事实来源（Source of Truth）：
  - `StaticRouter/View/Customized/ButtonStyle.swift` 中的 `RouterTheme`
  - 各页面组件样式（RouteList/SystemRoute/Settings/RouteEditSheet）

## 3. 视觉 Token

### 3.1 颜色

核心语义色（来自 `RouterTheme`）：

- Accent: `#2970E6`（主强调）
- Success: `#249E66`
- Warning: `#E8961C`
- Danger: `#D64238`

中性色使用策略：

- 背景与分层：优先使用系统材质与系统背景色（`windowBackgroundColor` / `underPageBackgroundColor` / `controlBackgroundColor`）。
- 边框与填充：用 `Color.primary.opacity(...)` 生成弱对比边框与浅底，保证浅色/深色模式均可读。

### 3.2 圆角与边框

- 小型操作按钮：圆角约 `7-8`
- 内容卡片：圆角约 `10-12`
- 边框透明度：弱边框优先（0.5~0.6 线宽），避免“粗描边”。

### 3.3 字体层级

- 页面主标题：`title3` ~ `title2` + `semibold/bold`
- 区块标题：`subheadline` + `semibold`
- 常规内容：`body/callout`
- 辅助信息：`caption/footnote` + secondary style
- IP/CIDR/网关值：`monospaced` 字体，提升扫描效率

## 4. 组件规范

## 4.1 顶部 Banner（状态提示）

- 仅用于“需要用户注意的状态变化”。
- 使用浅底语义色，不用高饱和纯色块。
- 文案 + 单操作按钮，结构保持单行优先。

## 4.2 路由表操作按钮

- 改为统一尺寸图标按钮（弱底 + 弱边框）。
- 颜色表达行为语义：编辑（accent）、分组（success）、删除（danger）。
- 避免旧版高饱和大块按钮并排导致视觉噪声。

## 4.3 系统路由工具栏

- 搜索框 / “仅我的路由” / 刷新按钮统一 `controlSize(.regular)`。
- 控件高度和权重一致，减少“拼凑感”。

## 4.4 侧边栏底部工具区

- 保持单层结构（`bar` 背景）+ 小尺寸 bordered 按钮。
- 避免多层容器叠加（材质 + 额外描边 + 外层背景）导致冗余。

## 4.5 设置页（通用/关于）

- 使用卡片化区块承载核心信息。
- 窗口高度为“按当前 Tab 内容自适应”，不再固定死高度。
- 操作按钮尽量并入信息卡片，减少上下断裂和空白。

## 4.6 添加/编辑路由弹窗

- 结构分区：目标网络 / 路由方式与网关 / 分组。
- 网关类型使用 segmented 选择，不使用 radio 纵向堆叠。
- 分组使用可点击标签卡片，支持快速多选并即时可视反馈。
- 输入错误统一使用语义红色边框 + 文本提示。

## 5. 动效与交互

- 动效时长：优先 `0.16s ~ 0.2s` 的短反馈。
- 仅在状态切换处使用轻动画（按钮按压、Tab 内容高度变化）。
- 避免会影响布局稳定性的缩放动画。

## 6. 可访问性与可读性

- 颜色不是唯一状态指示，保留图标/文本辅助。
- 主信息与次信息保持足够对比度。
- 交互组件保持系统语义（Button/Toggle/TextField），不以静态视图伪装可点击元素。
- 表格关键字段使用等宽字，避免数字与网段阅读错位。

## 7. 页面落地清单（新增页面时）

- 先定义“标题 + 工具栏 + 主内容 + 状态栏”四段式层级。
- 先复用 `RouterTheme`，避免新增孤立颜色。
- 可操作图标按钮统一尺寸与圆角。
- 对空状态、错误状态、禁用状态提供明确反馈。

## 8. 禁止项（Anti-patterns）

- 同一行出现多个高饱和 prominent 按钮竞争注意力。
- 不同工具栏控件尺寸不一致（尤其搜索/筛选/刷新）。
- 无语义的装饰性阴影、渐变、额外容器叠层。
- 固定窗口高度导致大面积空白或内容挤压。
